# BUILD MVP Mode - Production Implementation

**Command:** `do buildmvp`

## Purpose
Implement robust MVP code exactly as defined in approved `PLAN_MVP`, prioritizing quality, testability, comprehensive error handling, and production readiness.

## Core Principle
Build production-ready code with comprehensive error handling, testing, monitoring, and documentation. Quality over speed.

## Process Overview

1. **Verify Plan** - Ensure approved `PLAN_MVP` context available
2. **Implement Production Backend** - Robust functions with comprehensive validation
3. **Implement Production Frontend** - Polished UI with error boundaries
4. **Implement Comprehensive Testing** - Unit, integration, and E2E tests
5. **Implement Monitoring** - Logging, metrics, alerting
6. **Production Documentation** - User guides, technical docs, API docs

## Critical Requirements

### Production Quality Standards
- **Comprehensive Error Handling:** All error scenarios covered
- **Input Validation:** Strict validation at all boundaries
- **Performance Optimization:** Efficient queries, caching, bundling
- **Security Implementation:** Access control, audit trails, data protection
- **Monitoring & Observability:** Structured logging, metrics, alerting

### Testing Requirements
- **Unit Tests:** 90%+ coverage for business logic
- **Integration Tests:** All API endpoints and data flows
- **E2E Tests:** Complete user workflows
- **Performance Tests:** Load testing and benchmarks
- **Security Tests:** Vulnerability and penetration testing

## Production Standards

### Code Quality
- **TypeScript:** Strict typing, no `any` types
- **Error Handling:** Comprehensive try-catch, user-friendly messages
- **Performance:** Optimized queries, efficient algorithms
- **Security:** Input sanitization, access control, audit logging
- **Maintainability:** Clear code structure, comprehensive documentation

### Monitoring & Observability
- **Structured Logging:** Consistent log formats, appropriate levels
- **Metrics Collection:** Performance, usage, error metrics
- **Alerting:** Critical error and performance alerts
- **Audit Trails:** User action logging for compliance
- **Performance Monitoring:** Response times, resource usage
