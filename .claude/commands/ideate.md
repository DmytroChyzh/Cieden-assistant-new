# IDEATE Mode - Creative Solution Explorer

**Command:** `do inn`

## Purpose
Generate and evaluate **≥2 distinct, viable approaches** for implementing features. Explore solution space before committing to a specific path. Balance innovation with consistency and feasibility.

## Process Overview

1. **Context Review** - Review RESEARCH findings and existing patterns
2. **Brainstorm Approaches** - Generate ≥2 distinct implementation strategies
3. **Targeted External Research** - Validate novel patterns if needed
4. **Evaluate Approaches** - Create pros/cons comparison
5. **Recommend & Justify** - Select best approach with rationale

## Multi-Agent Opportunities 🚀

IDEATE mode has **exceptional potential** for parallel creative exploration:

### Parallel Approach Development
```
Agent 1: Leverage Existing System
├── Custom builders utilization
├── Existing pattern adaptation
├── Minimal architectural changes
└── Fast implementation focus

Agent 2: Alternative Innovation
├── New pattern exploration  
├── Modern UI/UX approaches
├── Advanced technical solutions
└── Future-proof architecture

Agent 3: Hybrid Approaches
├── Best-of-both solutions
├── Phased implementation paths
├── Migration-friendly options
└── Risk mitigation strategies
```

### Parallel Prototype Development
**High-value multi-agent opportunity:**

```
Agent 1: UI/UX Prototypes
├── Create mockups in app/prototypes/[feature]-concept-1/
├── Basic component structure
├── User interaction flows
└── Visual concept validation

Agent 2: Technical Prototypes  
├── Create backend POCs in app/prototypes/[feature]-tech-1/
├── Core logic validation
├── Data flow testing
└── Performance feasibility

Agent 3: Integration Prototypes
├── Create full-stack demos in app/prototypes/[feature]-full-1/
├── End-to-end flow validation  
├── Real data integration
└── User experience testing
```

### Implementation Strategy

1. **Parallel Concept Generation**
   - Each agent develops distinct approaches independently
   - Prevents groupthink and ensures diverse perspectives
   - Agents can build on different architectural philosophies

2. **Concurrent Prototype Building**
   - Multiple working prototypes created simultaneously
   - Quick feasibility validation for each approach
   - Visual/interactive comparison for user feedback
   - Prototype location: `app/prototypes/[feature]-[approach]/`

3. **Parallel External Research**
   - Agent 1: Technical research (libraries, patterns, APIs)
   - Agent 2: UX research (best practices, competitors, trends)
   - Agent 3: Integration research (existing system compatibility)

4. **Independent Evaluation**
   - Each agent evaluates approaches against different criteria
   - Technical feasibility vs. UX quality vs. maintainability
   - Results synthesized for comprehensive comparison

### Prototype Management
- **Creation:** `app/prototypes/[feature-name]-[approach-name]/`
- **Documentation:** Each prototype includes README with concept summary
- **Cleanup:** Non-winning prototypes archived or removed after decision
- **Evolution:** Winning prototype moves to main feature implementation

## Evaluation Criteria

### POC Focus
- Speed/Simplicity
- Concept validation
- Quick feedback capability

### MVP Focus  
- Robustness/Scalability
- Production readiness
- Comprehensive testing

### General Criteria
- User Experience quality
- Technical feasibility with current stack
- Consistency with existing patterns
- Maintainability and future evolution
