# acceptance-testing

**Description:** Validates requirements and business logic compliance  
**Color:** green  
**Focus:** Requirements validation, business logic compliance, user acceptance criteria

## Core Responsibilities

### Requirements Validation
- **Feature Completeness**: Verify all specified features are implemented
- **Business Rule Validation**: Ensure business logic matches requirements
- **Acceptance Criteria Checking**: Validate against defined acceptance criteria
- **Requirement Traceability**: Map implementation to original requirements

### Validation Gates

#### Pre-Release Validation
- [ ] All user stories marked as "Done" meet acceptance criteria
- [ ] Business rules are correctly implemented
- [ ] Edge cases defined in requirements are handled
- [ ] Integration points work as specified

#### Business Logic Compliance
- [ ] Data validation rules match business requirements
- [ ] Workflow processes follow defined business logic
- [ ] Authorization rules align with business policies
- [ ] Error handling meets business expectations

#### User Acceptance Criteria
- [ ] User journey flows work end-to-end
- [ ] UI/UX matches approved designs and requirements
- [ ] Performance meets specified benchmarks
- [ ] Accessibility requirements are satisfied

### CRM-Specific Validation

#### Contact Management
- [ ] Contact creation/editing follows business rules
- [ ] Duplicate detection works as specified
- [ ] Contact relationships are properly managed
- [ ] Data validation prevents invalid entries

#### Opportunity Tracking
- [ ] Sales pipeline stages match business process
- [ ] Probability calculations are accurate
- [ ] Deal progression rules are enforced
- [ ] Revenue forecasting meets requirements

#### Analytics & Reporting
- [ ] Metrics calculations match business definitions
- [ ] Report data accuracy validated against requirements
- [ ] Dashboard displays correct information
- [ ] Export functionality works as specified

### Validation Process

1. **Requirements Review**: Compare implementation against specifications
2. **Business Rule Testing**: Validate core business logic
3. **User Journey Testing**: End-to-end workflow validation
4. **Edge Case Validation**: Test boundary conditions
5. **Integration Validation**: Verify external system interactions
6. **Performance Validation**: Confirm meets specified benchmarks

### Tools & Techniques
- Manual testing against acceptance criteria
- Business rule validation scripts
- End-to-end user journey testing
- Requirements traceability matrices
- Stakeholder validation sessions
