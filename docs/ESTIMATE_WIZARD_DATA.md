# Estimate wizard — джерело даних і питання

## Звідки беруться цифри (скільки і за що берете гроші)

**Так, усі діапазони цін і термінів у майстрі оцінки взяті з вашої папки з естімейтами.**

- **Код:** `src/data/estimateRanges.ts` — масив `ESTIMATIONS` з полями `minPrice`, `maxPrice`, `timeline`, `projectType`, `complexity`.
- **Першоджерело:** дані з папки `src/data/Estimation file/`:
  - **real-estimations.json** — структура по типах проєктів (website, e-commerce, mobile-app, landing, redesign, dashboard, admin-panel) і складності (low, medium, high). Саме ці пари (тип + складність) і відповідні діапазони перенесені в `estimateRanges.ts`.
- Для опції **«Other»** (інший тип / своя відповідь) використовується узагальнений діапазон за складністю з тих самих даних: `getGenericEstimationRange(complexity)` бере мін/макс по всіх записах з такою складністю.

Якщо зміните цифри в `Estimation file/real-estimations.json`, потрібно оновити `src/data/estimateRanges.ts` (або підключити імпорт з папки з пробілом у назві).

## Скільки питань і які найголовніші

У дизайн-компаніях зазвичай уточнюють:

1. **Тип роботи:** з нуля чи редизайн (у нас: «What do you need?» + опція «Other»).
2. **Тип продукту:** сайт, e-commerce, додаток, дашборд, лендінг, адмінка тощо (у нас: список + «Other» з полем для опису).
3. **Складність/обсяг:** low / medium / high (кількість екранів, ролей, інтеграцій).
4. **Терміни:** бажані дедлайни (у наших даних — у кожному рядку є `timeline`; окремого питання в майстрі поки немає).
5. **Бюджет:** орієнтовний діапазон клієнта (у майстрі не питаємо — показуємо наш діапазон).

Зараз у майстрі **3–4 кроки** (залежно від відповіді «з нуля» / «редизайн» / «інше»):

- Питання 1: що потрібно (з нуля / редизайн / інше + опційно свій опис).
- Питання 2 (якщо «з нуля»): тип проєкту (включно з «Other» + опційно свій опис).
- Питання 3 (або 2 для редизайн/інше): складність.
- Крок 4: результат (діапазон з ваших даних + підказка зв’язатися з менеджером).

Можна згодом додати ще питання (наприклад, терміни, кількість екранів), якщо з’являться відповідні дані в Estimation file і логіка розрахунку.
