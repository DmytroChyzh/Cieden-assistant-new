# План: Топ-50 питань клієнтів + інтерактивні картки та правила асистента

**Відповіді на ці питання (реальні факти Cieden):** див. **`docs/CIEDEN_KNOWLEDGE_TOP50.md`**. Інформація взята з cieden.com, /about, /services, /pricing-models та FAQ. Асистент має відповідати **своїми словами** на основі цього документу; не копіювати відповіді слово в слово.

---

## Мета
- Клієнт отримує **живу** відповідь текстом; ніде не має бути жорстко прописаних "Відповідь 1", "Відповідь 2".
- Де доречно — асистент **доповнює** відповідь інтерактивною карткою (портфоліо, ціни, процес, наступний крок).
- Всі 50 питань покриті: частина — лише текстом, частина — текст + картка.

---

## 1. Поточний стан (що вже є)

### Інструменти та картки
| Інструмент | Картка | Статус |
|------------|--------|--------|
| `show_cases` | CasesGrid (сітка кейсів) | ✅ Є |
| `show_best_case` | BestCaseCard (один кейс, напр. Sitenna) | ✅ Є |
| `show_case_details` | CaseDetails(id) | ✅ Є в UI; перевірити реєстрацію в bridge + ElevenLabs |
| `show_engagement_models` | EngagementModelsCard | ✅ Є |
| `generate_estimate` | EstimateSummaryCard | ✅ Є |
| `open_calculator` | Зараз = EngagementModelsCard | ⚠️ Можна залишити або окрема картка "калькулятор" |

### Чего немає (треба додати)
- **About Cieden** — хто ми, послуги, індустрії, design vs dev. Одна картка.
- **Process & Timeline** — етапи проєкту, терміни, команда, комунікація. Одна картка.
- **Getting Started** — перший крок, як почати, "записатися на дзвінок" / "заповнити бриф". Картка з 1–2 CTA.

## 2. Як мають виглядати нові картки (правила дизайну)

- **Стиль:** той самий, що в існуючих Cieden-карток: темний фон, glass (backdrop-blur, напівпрозорий bg), `rounded-3xl`, тонка обводка `ring-1 ring-white/10`, без жорстких border.
- **Структура:**
  - Заголовок (коротко).
  - 1–2 речення або буллети / нумеровані кроки.
  - За потреби — одна або дві кнопки: "Book a call", "Fill brief", "See portfolio" тощо (посилання або `onUserAction`).
- **Контент:** тільки факти від Cieden (тексти підготуєте ви; в коді — пропси або константа з одним джерелом правди).
- **Мобільний вигляд:** картки в чаті вже адаптивні; нові — так само, без окремого макету.

Конкретно:
- **About Cieden card:** блоки "Who we are", "Services", "Industries", "Design vs development" — короткі абзаци або списки.
- **Process & Timeline card:** етапи (Discovery → Design → Handoff) з орієнтовними термінами; "Team: PM + Lead Designer + …"; "Communication: Slack, Trello".
- **Getting Started card:** 2–3 кроки (наприклад: 1) Розмова 2) Бриф 3) Пропозал) + CTA "Book a call" / "Fill brief".

## 3. Правила для асистента (що прописати в ElevenLabs / системному промпті)

Нижче — текст, який можна вставити в інструкції агента (адаптувати під ваш тон).

### 3.1 Загальна поведінка
- Ти — живий представник Cieden. Відповідай **як людина**: коротко, по суті, без шаблонів типу "Відповідь 1", "Відповідь 2".
- Мову підлаштовуй під клієнта (якщо пишуть українською — відповідай українською, англійською — англійською), якщо не вказано — можна англійською за замовчуванням.
- Якщо питання про портфоліо, ціни, процес або "як почати" — **додатково** викликай відповідний інструмент (show_cases, generate_estimate, show_engagement_models, show_process, show_getting_started тощо), щоб у чаті з’явилась картка. Відповідь текстом все одно дай коротко своїми словами, не кажи буквально "зараз покажу картку".
- Якщо питання про факти (де офіс, чи аутсорс, скільки років на ринку) — відповідай **тільки текстом**, без картки.
- Не перевантажуй: одна думка — 1–2 речення; якщо показуєш картку — можна одне речення + картка.

### 3.2 Коли який інструмент викликати (орієнтир)
- **Портфоліо / кейси / приклади робіт** → `show_cases`. Якщо про "найкращий" або "один яскравий кейс" → `show_best_case`. Якщо клієнт назвав галузь або хоче деталі одного кейсу → `show_case_details` (з параметром id або filter, якщо є).
- **Ціни / орієнтовна вартість / бюджет** → коротка текстова відповідь + `generate_estimate` (можна з параметрами productType, complexity, scope — якщо клієнт щось із цього сказав).
- **Моделі співпраці / ретейнер / підписка** → `show_engagement_models`.
- **Процес, етапи, терміни, хто в команді** → `show_process` (нова картка Process & Timeline).
- **Хто ми, що робимо, послуги, індустрії** → `show_about` (нова картка About Cieden).
- **Як почати, перший крок, записатися на дзвінок** → коротка відповідь + `show_getting_started` (нова картка з CTA).

### 3.3 Чего уникати
- Не пиши "Я показую вам картку нижче" / "Ось інтерактивний блок" — просто дай текст і викликай tool.
- Не давай довгих списків з 50 питань у одній відповіді; якщо клієнт питає "що ви робите?" — коротко + картка About, без переліку всіх 50 питань.
- Не викликай два інструменти в одній відповіді, якщо це не потрібно (наприклад, портфоліо + ціни окремими повідомленнями — ок; в одному повідомленні — один фокус).

## 4. Розподіл 50 питань: текст vs картка

**Логіка:** Одна картка часто покриває кілька питань (наприклад 16–25 — це одна картка «Процес», але в ній є секції: процес, терміни, швидкість старту, ітерації, команда, комунікація, discovery). Так клієнт отримує одну інформативну картку за темою, а не окрему картку на кожне питання. Нижня таблиця і наступний підрозділ деталізують, які саме питання відповідають якій картці.

| # | Питання | Тип відповіді | Інструмент / картка |
|---|---------|----------------|----------------------|
| 1 | Що робить Cieden? Про компанію та послуги. | Текст + картка | show_about |
| 2 | З якими індустріями працюєте? | Текст + картка | show_about |
| 3 | Тільки дизайн чи й розробка? | Текст + картка | show_about |
| 4–7 | Типовий клієнт, локація, віддалено, скільки років | Текст | — |
| 8 | Покажіть портфоліо або кейси | Текст + картка | show_cases |
| 9 | Найкращий / найрелевантніший кейс | Текст + картка | show_best_case |
| 10 | Кейси в нашій ніші / індустрії | Текст + картка | show_cases або show_case_details |
| 11–15 | Метрики успіху, before/after, приклади по типу продукту, ребрендинг, тривалість подібного проєкту | Текст + картка | show_cases / show_case_details |
| 16 | Який дизайн-процес і терміни? | Текст + картка | show_process |
| 17–25 | Швидкість старту, ітерації, команда, комунікація, discovery, час клієнта, бриф, етапи, research | Текст + картка | show_process |
| 26 | Скільки коштує типовий проєкт? Орієнтовна оцінка | Текст + картка | generate_estimate |
| 27–34 | Моделі співпраці, формування ціни, базовий пакет, UI/UX без розробки, мін. бюджет, оплата, пропозал, що збільшує вартість | Текст + картка | show_engagement_models / generate_estimate |
| 35 | Як почати проєкт з Cieden? Перший крок | Текст + картка | show_getting_started |
| 36–40 | Тільки дизайн чи реалізація, контракт/NDA, онбординг, записатися на дзвінок, бриф/анкета | Текст + картка | show_getting_started |
| 41–50 | Після здачі, підтримка, файли/гайди, ретейнер, формат макетів, прототипи, design system, a11y, тестування, тривалість проєкту | Текст + картка | show_support |

Підсумок: відповіді **завжди** текстом (гнучко), картки — **додатково** там, де це корисно (портфоліо, ціни, процес, "хто ми", "як почати", підтримка).

**Скільки питань інтерактивні (з карткою):** питань **1–3** (About), **8–10** (кейси), **11–15** (кейси/деталі), **16–25** (процес), **26–34** (ціни/моделі), **35–40** (як почати), **41–50** (підтримка) — тобто **46 з 50** мають відповідну картку. Лише **4–7** (типовий клієнт, локація, віддалено, скільки років) — тільки текст. Тобто **більше половини** (насправді ~92%) питань можна підкріпити інтерактивною карткою.

### Деталізація блоків (що саме входить у кожен діапазон)

Одна картка може покривати кілька питань: у ній є секції під кожну тему. Нижче — розпис по номерах, щоб було зрозуміло, за яке питання яка картка.

| Блок | № питань | Про що питання (коротко) | Картка |
|------|----------|--------------------------|--------|
| About | 1 | Що робить Cieden, компанія, послуги | show_about |
| | 2 | З якими індустріями працюєте | show_about |

| Кейси | 8 | Портфоліо / кейси | show_cases |
| | 9 | Найкращий кейс | show_best_case |
| | 10 | Кейси в нашій ніші | show_cases / show_case_details |
| | 11–15 | Метрики, before/after, приклади, ребрендинг, тривалість проєкту | show_cases / show_case_details |
| **Процес** | **16** | **Який дизайн-процес і терміни?** | **show_process** |
| | **17** | **Як швидко можете почати?** | **show_process** |
| | **18** | **Скільки ітерацій/правок?** | **show_process** |
| | **19** | **Хто вестиме проєкт? Скільки людей?** | **show_process** |
| | **20** | **Як будете комунікувати?** | **show_process** |
| | **21** | **Як розумієте наш бізнес на початку? (discovery)** | **show_process** |
| | **22–25** | **Час клієнта, бриф, етапи, user research** | **show_process** |
| Ціни | 26 | Скільки коштує типовий проєкт | generate_estimate |
| | 27–34 | Моделі співпраці, формування ціни, бюджет, оплата, пропозал | show_engagement_models / generate_estimate |
| Як почати | 35 | Як почати проєкт? Перший крок | show_getting_started |

Тобто для **16–25** одна картка **«Process, team & communication»** — але у ній окремі секції: процес і етапи, терміни, швидкість старту, ітерації, команда, комунікація, discovery. Так клієнт бачить відповіді на всі ці підпитання в одному місці, без окремої картки на кожне.

## 5. План робіт (етапи)

### Фаза 1: Підготовка контенту та правил (без коду)
- [ ] Зібрати фінальні тексти для карток **About Cieden**, **Process & Timeline**, **Getting Started** (в одному документі або таблиці).
- [ ] Зафіксувати посилання/форми для CTA: "Book a call", "Fill brief" (щоб потім підставити в картки).
- [ ] Перенести блок "Правила для асистента" (розділ 3) в інструкції ElevenLabs-агента; при потребі скоротити під ліміт символів.
- [ ] Переконатися, що `show_case_details` зареєстрований у bridge і в ElevenLabs (якщо хочете "один кейс по id/галузі").

### Фаза 2: Нові картки в коді
- [ ] **About Cieden:** компонент (наприклад `AboutCiedenCard`) з секціями: хто ми, послуги, індустрії, design vs dev. Стиль як у SalesUi (glass, rounded-3xl).
- [ ] **Process & Timeline:** компонент `ProcessTimelineCard` — етапи, терміни, команда, комунікація.
- [ ] **Getting Started:** компонент `GettingStartedCard` — 2–3 кроки + кнопки CTA (посилання або подія для трекінгу).
- [ ] Експорт з `SalesUi.tsx` (або окремого модуля) і підключення в `ToolCallMessageRenderer`.

### Фаза 3: Інструменти та проводка
- [ ] Додати в `toolBridge`: типи параметрів і хендлери для `show_about`, `show_process`, `show_getting_started` (параметри мінімальні або порожні).
- [ ] Зареєструвати ці три tools у `createClientTools` (elevenLabsTools) і в `actionHandlers` на сторінці voice-chat.
- [ ] У `ToolCallMessageRenderer` для `show_about` → `<AboutCiedenCard />`, `show_process` → `<ProcessTimelineCard />`, `show_getting_started` → `<GettingStartedCard />`.
- [ ] У ElevenLabs dashboard додати опис нових tools і в інструкціях вказати, коли їх викликати (згідно з розділом 3.2).

### Фаза 4: Топ-50 і quick prompts
- [ ] Переконатися, що empty state і QuickActionsDrawer містять питання з топ-50 (вже є 6; при бажанні додати ще 2–4 з розділу "Getting Started" / "Process").
- [ ] Опційно: один Markdown-файл з усіма 50 питаннями та позначкою "text only" / "text + card" для внутрішнього використання та оновлення інструкцій агента.

### Фаза 5: Тестування та правки
- [ ] Пройти список з 50 питань вручну (голос + текст): перевірити, що відповіді живі, без "Відповідь 1", і що картки з’являються там, де задумано.
- [ ] Підправити копірайти в картках і формулювання в інструкціях агента за результатами.

## 6. Чи можемо все це зробити

- **Так.** Інфраструктура вже є: tools → bridge → actionHandlers → ToolCallMessageRenderer → компоненти карток. Потрібно:
  - три нові компоненти карток (About, Process, Getting Started);
  - три нові tools і їх проводка;
  - оновлення інструкцій агента в ElevenLabs і (за бажанням) один документ з 50 питаннями та правилами.

Після виконання плану асистент буде гнучко відповідати текстом на всі 50 питань і там, де доречно — показувати інтерактивні картки без жорсткої нумерації відповідей.
